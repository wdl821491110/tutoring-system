## 🔴 新发现 Bug：电脑端数据丢失（任务 13）

### Bug 描述
电脑端（PyInstaller exe）打开后，新增数据后，过段时间再打开，数据就丢失了。

### 架构分析
当 `USE_PROXY = True`（默认值）且非云端环境时：
1. 所有业务 API 请求（除登录/鉴权外）都会被 `handle_local_proxy_or_cors()` 拦截，通过 `_proxy_to_cloudbase()` **代理到 Render 云端**。
2. 本地 exe 不直接读写本地 SQLite 数据库，所有读写都走云端 Render。
3. 本地 exe 的 `g.db = get_db()` 连接的是 `exe同级/data/tutoring.db`，但**只有备份/恢复等管理操作才用到这个本地数据库**。

### 根本原因

**核心问题：`trigger_cloud_backup()` 永远不会被触发。**

1. `add_cors_headers()` 函数（第242行）定义了写操作后触发云端备份的逻辑
2. **但从未被注册为 `@app.after_request`**（代码中搜索不到 `app.after_request` 注册记录）
3. 所以 `trigger_cloud_backup()` 从不会被调用

### 影响
- 本地 exe 的所有增删改操作虽然能成功代理到 Render 云端保存
- 本地 `exe/data/tutoring.db` 从未被写入任何数据
- 如果 Render 服务停止或清理数据，本地数据就"丢失"了
- 用户看到的数据来自 Render 云端代理，但没有任何本地持久化保障

### 修改计划

#### 任务 13-1: 注册 `add_cors_headers` 为 `after_request`
**文件**: `app.py`
**位置**: 第259行后
**修改**: 添加 `app.after_request(add_cors_headers)` 注册

#### 任务 13-2: 修复 `trigger_cloud_backup` 的触发条件
**文件**: `app.py`
**问题**: 当前仅在 `TCB_API_KEY` 配置时才触发云备份

#### 任务 13-3: 增加代理失败时的本地回退机制

**问题**: `_proxy_to_cloudbase()` 失败时直接返回 502 错误
**修改**: 当 Render 不可用时，尝试回退到本地数据库操作

#### 任务 13-4: 增加数据一致性日志
**文件**: `app.py`
